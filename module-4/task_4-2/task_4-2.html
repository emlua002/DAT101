  6. In task 8, we will use a "callback" routine when sorting. Explain how such a "callback" routine works.

    7. A callback is a function you give to another function so it can use it when needed. In task 8, the .sort() function uses a callback to compare two numbers at a time and decide their order. The callback returns a number: a negative number means the first number comes first, positive means the second number comes first, and zero means they stay in the same order. A callback lets you customize or extend a functions behavior without changing its code.

  7. What happens in the sorting routines you have created? Try to explain how the array can figure out if an element should come before or after another. The best way to answer this is to actually print both parameters each time the sorting routine runs.

    8. When .sort() is used with a callback, the array compares elements two at a time. Each time, the callback recieves two elements (a and b) and returns: - Negative - a comes before b - Positive - b comes before a - Zero - no change in order The array doesn't just compare elements once, it keeps calling the callback for different pairs of elements until everything is in the correct order. Printing the parameters inside the callback shows which elements are being compared and how the return value decides their order.
