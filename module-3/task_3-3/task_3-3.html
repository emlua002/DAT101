1. What is the difference between invoking and creating a function?

    2. Creating a function is when you give it a name, parameters and a body. it doesn't run yet. Invoking a function is when you actually call it to run the code inside it.

  2. Explain the syntax regarding semicolons and code blocks.

    3. Semicolons are used to mark the end of a statement, while code blocks are defined by curly brackets { } and define groups of statements that belong together. For example functions, loops conditionals etc.

  3. What happens when we try to store values from a function when it doesn't return any?

    4. If a function doesn't return a value, it automatically returns "undefined". This is because nothing was returned.

  4. How do we separate parameters from each other when creating a function?

    5. When we are creating a function, we separate parameters from each others by using commas. For example -> function myFunction(parameter1, parameter2)

  5. Explain how parameter names are related during the invocation and creation of a function

    6. When we create a function, we define parameter names, which works as placeholders for the values. When we invoke the function, we give it arguments that match the parameter names in both position and type. That is why it is a good practice to have descriptive parameter names.

  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. If you define a parameter without giving an argument for that parameter, it will automatically be assigned the value "undefined". This is because JavaScript does not give an error for missing parameters.

  7. Explain what happens when we use recursive functions.

    8. A recursive function is one that calls itself. It will keep going until it reaches a "base case". Which is a condition that stops the recursion. Each function call adds a new layer to the call stack. If we don't have a base case, the function will continue to call itself forever, leading to a stock overflow error.
